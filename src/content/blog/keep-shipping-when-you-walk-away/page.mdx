## The problem

Simon Willison fires up four parallel agents and is wiped out by 11 AM. I've felt the same thing. The tools are powerful, our skills are more in demand than ever, and yet we're fried before lunch. Context switching was already expensive — agents made it worse because now you're managing multiple streams of autonomous work on top of everything else.

The bottleneck is you. Agents scale infinitely. Your attention degrades under load. If you scale linearly with the tools, you burn out in 18 months.

## The opening demo

I opened with a bug from a Slack bot I built at WorkOS — the sentence case enforcer was mangling acronyms like SCIM and SSO. Instead of context-switching into the codebase myself, I gave Claude Code MCP access to Slack and Linear. It found the bug report, fixed the code, deployed, posted a test message, read its own output, and confirmed the fix. I came back to a completed, verified loop.

## Four layers

I walked through four capabilities that make this sustainable.

### Signal agents

Slack and Linear MCP plugged into Claude Code. Messages flow in, Claude classifies them — action needed, noise, duplicate of an existing ticket — and auto-creates Linear tickets for the real work. In one demo, 16 incoming Slack messages got distilled to 2 new tickets and 1 dedup. I never open Slack. I only see the delta.

### Voice-first flows

I've been using <Link href="/blog/walking-and-talking-with-ai">WisprFlow for voice-first development</Link> for over a year, hitting 184 WPM compared to ~90 WPM typing. I showed the dispatch advantage: speak 3 instructions to 3 agents simultaneously. Total dispatch time is about 9 seconds. A keyboard-only developer is still typing the first task.

### Remote control

With the `--remote-control` flag, agents work on local files while I'm on the trail. I check progress on my phone over LTE and redirect them with voice as new ideas come. The desk is optional. I also showed <Link href="/blog/handwave">Handwave</Link>, the watchOS app that lets me control sessions from my wrist.

### The system improves itself

Every week, a scheduled task scans all my Claude Code JSONL session logs from the previous week. It extracts patterns — tasks done more than once, repeated instructions, workflows that could be automated. The output is a list of new skills to build and inefficiencies I missed. One week it flagged that I'd reformatted blog images for CDN upload four separate times without noticing.

## Verification gates

Speed without safety is recklessness. Three tiers:

1. **Lint and build** — TypeScript compilation, ESLint.
2. **Browser use** — The agent launches the app, takes screenshots, visually confirms the output.
3. **Critic passes** — CLAUDE.md rules, custom linting, project-specific constraints that catch subtle errors automated tests miss.

## The Oura Ring integration

I demoed the <Link href="/blog/connect-oura-ring-to-claude-desktop-with-mcp">Oura Ring MCP integration</Link> I built. It exposes my sleep and HRV data to Claude Code. When I'm planning my day, Claude has access to my physical state. If I slept poorly, it adjusts. You can ignore it — but at least you thought about it.

## The intentional path

I closed with two paths. The default: stack more loops, run at 120%, burn out. The intentional path: design workflows that protect your attention, ship more while moving more and sleeping more, sustain it for years.

The tools are powerful. Your nervous system is ancient. Find your developer balance.
