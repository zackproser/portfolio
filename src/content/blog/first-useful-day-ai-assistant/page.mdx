I run an [always-on AI assistant](/blog/building-always-on-ai-assistant) on a mini PC under my desk. I'm in one of the most intensive phases of my life right now, and I genuinely need something that takes work off my plate. I need real infrastructure that handles actual work.

Most days, it's been promising but not quite there. This was the first day that changed.

## Sunday dig-out

It started with a Linear ticket: "Sunday Dig-Out." One task with a wall of subtasks — open PRs, bug fixes, affiliate content, expert network applications, flight booking, new assistant capabilities. I dropped it into the conversation and said: here's the day. Let's go.

## Flight planning as a conversation

I needed to fly to San Francisco for a workshop on Thursday. I'm an AAdvantage member, prefer American nonstops, and a nor'easter was bearing down on the northeast.

So I just started talking. "Arrive early Wednesday to decompress." Then: "Actually, maybe Tuesday night after the kids." Then: "Wait, is BWI safe to connect through?"

The assistant kept up because it had the tools — flight data from Kiwi MCP, weather from the NWS, my budget and schedule constraints. When I worried about the storm, it pulled the actual hazardous weather outlook and showed me it was tracking well north of Charlotte.

Thirty minutes of thinking out loud → clear plan, Tuesday evening nonstop. An assistant with the right API keys converges on decisions instead of looping indefinitely.

## Building tools mid-conversation

The email integration was broken — Gmail API only showed the last handful of emails. Useless for a real sweep.

So I told the assistant to build a fix. It wrote a Go CLI that mirrors my entire Gmail inbox into local SQLite with FTS5 full-text search. I critiqued the code from another terminal with Claude Code, sent back a bug report with six issues, it fixed all of them.

Twenty minutes later: 205,023 emails indexed locally, 17 years of history, searchable in milliseconds. The technical details are in [a separate post](/blog/gmail-mirror). The point is I built the tool *as part of my day*, in conversation, without breaking flow. The assistant recognized it needed better capabilities, we built them, and immediately used them.

## The $30K discovery

Right after syncing, I searched for expert network platforms — GLG, Tegus, AlphaSights, Guidepoint, Coleman Research. Companies that pay $500/hr for phone consultations about things you already know.

Sixty seconds across 205K messages:

- **65 consultation opportunities** in the past year
- **Only 5 completed**
- **~$30,000 left on the table**

I saw most of the invitations. I just couldn't act fast enough — these have 24-48 hour response windows. When you're managing everything else, a $500/hr phone call that requires a 10-minute response somehow doesn't happen.

That search only worked because of the tool we'd built that same day. Every conversation potentially creates a new capability.

## Linear as shared state

Having a single Linear ticket as the day's source of truth meant the assistant could hold context across every pivot. I jumped from flights to email infrastructure to blog writing to weather monitoring — three direction changes, multiple rabbit holes — and the thread held.

The memory is persistent. It knows what happened this morning, what we decided last week, what's on the task board. The state lives in the system, not in my head.

## The compounding effect

Every time we hit a wall, the gap becomes a permanent new capability:

- **Email search broken** → built gmail-mirror. Now it searches 205K emails in milliseconds, forever.
- **Needed flight data** → wired up the Kiwi MCP server. Reliable flight search without a browser.
- **Wanted affiliate analytics** → connected the Dub.co API. Tracks which articles drive actual conversions.

Each is usually a script and an API key, saved as a skill that persists across sessions. Six months ago, it could barely read my email. Now it searches 17 years of history, pulls flights, tracks commissions, queries bank accounts, manages tasks, checks sleep data, and monitors weather — all in one conversation where I'm just thinking out loud.

## Demo vs. daily driver

You need enough tools working reliably enough, across enough of your life, with enough shared context, that using the system costs less cognitive load than doing things yourself. Reliability × breadth × context.

Today crossed that line.

By 9 PM I'd forgotten half of what I'd shipped. The assistant posted its evening wind-down unprompted — cross-referencing Linear, email, and GitHub to compile everything: flights booked, gmail-mirror deployed, the $30K discovery, SEO fixes merged, articles delivered. It laid out tomorrow's calendar, flagged unanswered threads, and reminded me to take a new medication I'd already forgotten about.

That medication reminder alone justified the system. I would have missed my first dose of a sleep med I'd spent weeks getting prescribed.

The tools we built that morning were already feeding the evening summary, making it sharper than anything it could have produced 24 hours earlier. The system gets better every time you use it — and it keeps getting better whether you're watching or not.
