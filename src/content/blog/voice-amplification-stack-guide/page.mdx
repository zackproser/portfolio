## Layer 1: WisprFlow (Voice Input)

This is the foundation. Everything else builds on being able to speak instead of type.

### Installation

1. Download <AffiliateLink product="wisprflow" campaign={campaign}>WisprFlow</AffiliateLink> from [wisprflow.com](https://wisprflow.com)
2. Install the macOS app (Windows and mobile coming — currently Mac only)
3. Grant microphone and accessibility permissions when prompted
4. Set your activation hotkey (I use `Fn` — it's easy to hold with my left pinky)

### Two Modes You Need to Know

**Hold-to-talk:** Hold your hotkey, speak, release. Text appears wherever your cursor is. Use this for quick prompts, Slack messages, emails, terminal commands.

**Hands-free mode:** Double-tap your hotkey. WisprFlow stays active — you can walk around, gesture, pace. Speak for up to six minutes continuously. Double-tap again to stop. Use this for long prompts, brain dumps, and architecture discussions with yourself.

### Pro Tips

- **Don't slow down or enunciate carefully.** WisprFlow's AI handles natural speech, ums, false starts. Just talk normally.
- **Speak punctuation only when needed.** It usually figures out periods, commas, and question marks from context. Say "new paragraph" when you want one.
- **Context-aware:** It knows whether you're in a terminal, IDE, browser, or chat app and adjusts formatting accordingly.
- **Train yourself on Day 1-3.** It feels weird talking to your computer. By Day 3, it's natural. By Day 7, typing feels painfully slow.

## Layer 2: Claude Code (AI Execution)

With WisprFlow handling input, Claude Code becomes your hands. You speak what you want built; it builds it.

### Installation

```bash
# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Or if you prefer npx (no install)
npx @anthropic-ai/claude-code
```

You'll need an Anthropic API key. Set it:
```bash
export ANTHROPIC_API_KEY=your-key-here
```

### The Core Workflow

1. **Open a terminal** in your project directory
2. **Run** `claude` to start a session
3. **Hold your WisprFlow hotkey and speak:** "Look at the auth module and refactor the token validation into its own file. Keep the existing tests passing and add a new test for expired tokens."
4. **Release the hotkey.** Your spoken instruction appears as text in the Claude Code prompt.
5. **Press Enter.** Claude Code reads your codebase, plans the changes, and executes them.
6. **Review the diff,** approve or refine with another spoken instruction.

That's it. You just wrote code by talking.

### Running Multiple Sessions

This is where the voice speed really compounds. Open 3-4 terminal tabs, each with a Claude Code session in a different directory or on a different task:

- Tab 1: Backend API changes
- Tab 2: Frontend component updates
- Tab 3: Test suite
- Tab 4: Documentation

Rotate between tabs, speaking instructions into each one. While Claude works on Tab 1, you're already speaking to Tab 2. The horse race visualization below shows the difference — one person speaking to three agents simultaneously vs. someone typing one task:

## Layer 3: Phone Access (Go Mobile)

This is what makes the stack truly location-independent. You don't need to be at your desk.

### Option A: Claude Code via SSH (Any Phone)

If your dev machine is accessible via SSH (local network, Tailscale, or a VPS):

1. **Install a terminal app** on your phone — Termius, Blink, or the built-in Terminal on iOS 18+
2. **SSH into your dev machine:** `ssh you@your-machine`
3. **Start or resume a Claude Code session:** `claude` or reattach to a tmux session
4. **Use your phone's dictation** (hold the mic button on the keyboard) to speak prompts

This works surprisingly well. Phone dictation isn't as good as WisprFlow, but it's good enough for steering agents and giving high-level directions.

### Option B: Handwave (Apple Watch)

I built <Link href="/blog/handwave">Handwave</Link> — a watchOS app that connects to Claude Code sessions via Bonjour protocol. From my Apple Watch, I can:

- See all active Claude Code sessions
- Listen to the latest response from any session
- Speak a new direction and send it
- Resume sessions from anywhere

This is the setup I use on walks. AirPods for audio, Watch for control, phone in pocket as backup.

### Option C: Cursor on iPad/Phone (Coming)

Cursor's mobile app is in development. When it ships, you'll be able to run Cursor's AI features directly from a tablet or phone with voice input. Keep an eye on this.

### The Walking Setup

My actual gear for mobile development:

- **AirPods Pro** — noise cancellation + microphone for voice input
- **Apple Watch** with Handwave — checking and steering agents
- **iPhone** in pocket — SSH terminal for when I need to see code
- **WisprFlow on Mac at home** — sessions waiting for input via SSH

I walk 3-5 miles while "coding." The movement triggers diffuse-mode thinking, and I can act on ideas immediately by speaking to an agent.

## Putting It All Together

Here's what a typical day looks like with the full stack:

**8:30 AM** — Morning standup. Granola captures notes. I'm fully present, asking questions, not typing.

**8:45 AM** — Post-standup. I speak follow-up messages to my team via WisprFlow into Slack. 30 seconds.

**9:00 AM** — Deep work. Three Claude Code sessions open. I speak architecture decisions into one, implementation details into another, test cases into a third. Rotating between them.

**10:30 AM** — Walk break. I grab my AirPods and head outside. From my watch, I check on the three sessions. One needs a decision — I speak the answer. Another finished — I review the diff on my phone and approve.

**11:00 AM** — Client call. Granola captures everything. I'm fully present. After the call, I speak a summary email via WisprFlow. Sent in 45 seconds.

**11:15 AM** — Back to deep work. I do a brain dump about the client's requirements (6-minute verbal ventilation), Claude Code organizes it into a task plan, and starts on the first item while I review the morning's completed work.

**Total keystrokes for the day: maybe 200.** All for reviewing diffs and approving changes. Everything else was voice.
