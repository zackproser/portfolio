Vue.js powers a significant portion of the modern web — everything from small interactive widgets to enterprise dashboards built with Nuxt. Like all JavaScript frameworks, Vue renders its content in the browser. Send a plain HTTP request to a Vue SPA and you get back an empty mounting div, not the content you came for.

Scraping Vue and Nuxt sites requires the same fundamental approach as any JavaScript-rendered page: execute the code, wait for the DOM to stabilize, then extract from the result. But Vue and Nuxt have specific patterns that are useful to understand.

## How Vue Renders Content

Vue uses a virtual DOM that computes and applies DOM changes in batches. When a Vue component fetches data from an API, it:

1. Mounts with empty or loading state
2. Makes the data fetch asynchronously
3. Updates reactive state when data arrives
4. Re-renders the affected component tree

A scraper that grabs the DOM immediately after page load catches step 1 — loading spinners and empty lists. You need to wait for steps 2-4 to complete.

**Nuxt.js adds server-side rendering.** Nuxt can pre-render pages on the server, sending real HTML content that Vue then "hydrates" on the client. Nuxt's Universal Mode pages look like normal HTML to scrapers — the content is present in the initial response. But even SSR Nuxt pages have client-side components that load asynchronously after hydration.

## Basic Vue Site Scraping with Firecrawl

<AffiliateLink product="firecrawl" campaign={campaign}>Firecrawl</AffiliateLink> handles Vue's async rendering by running a real headless browser and waiting for the JavaScript lifecycle to complete:

```javascript
import Firecrawl from '@mendable/firecrawl-js'

const app = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY })

// Scrape a Vue SPA
const result = await app.scrapeUrl('https://vue-app.com/catalog', {
  formats: ['markdown'],
  waitFor: 2500, // Give Vue time to fetch data and re-render
})

console.log(result.markdown)
// Returns fully rendered catalog content, not loading placeholders
```

The `waitFor` value depends on the app. A Vue component that fetches from a fast local API might need 500ms. An app hitting multiple external APIs before rendering might need 4-5 seconds. Start at 2000ms and tune based on what you get back.

## Scraping Nuxt.js Applications

Nuxt sites need slightly different handling depending on their rendering mode:

```javascript
// Nuxt Universal Mode (SSR + hydration)
// Content is in initial HTML — shorter wait, mainly for hydration effects
const ssrPage = await app.scrapeUrl('https://nuxt-blog.com/posts/my-article', {
  formats: ['markdown'],
  waitFor: 1000, // Shorter — content already server-rendered
})

// Nuxt SPA Mode (client-side only, like plain Vue)
// Needs full wait for data fetching and rendering
const spaPage = await app.scrapeUrl('https://nuxt-dashboard.com/analytics', {
  formats: ['markdown'],
  waitFor: 3000,
})

// Nuxt with heavy Pinia store hydration
// Store rehydration can delay component renders
const storeHeavyPage = await app.scrapeUrl('https://nuxt-ecommerce.com/cart', {
  formats: ['markdown'],
  waitFor: 4000,
  actions: [
    { type: 'wait', milliseconds: 1500 }, // Extra stability buffer
  ]
})
```

## Targeting Vue Router Routes

Vue Router manages navigation in JavaScript. When a Vue app uses hash routing (`/#/products`) or HTML5 history routing (`/products`), Firecrawl's crawler follows these links correctly — both patterns result in valid URLs that Firecrawl can request and render:

```javascript
// Crawl all routes in a Vue SPA
const crawlResult = await app.crawlUrl('https://vue-shop.com', {
  limit: 50,
  scrapeOptions: {
    formats: ['markdown'],
    waitFor: 2500,
  },
  // Vue Router uses standard links — Firecrawl follows them automatically
  includePaths: ['/products/*', '/categories/*'],
  excludePaths: ['/admin/*', '/checkout/*'],
})

console.log(`Scraped ${crawlResult.data.length} Vue routes`)
```

## Structured Extraction from Vue Components

Vue's component-based architecture often creates predictable DOM patterns. Firecrawl's LLM-powered extraction handles these patterns without requiring custom selectors:

```javascript
import { z } from 'zod'

const ListingSchema = z.object({
  items: z.array(z.object({
    title: z.string(),
    price: z.number().optional(),
    category: z.string().optional(),
    url: z.string().optional(),
  }))
})

// Extract structured data from a Vue listing page
const result = await app.scrapeUrl('https://vue-marketplace.com/listings', {
  formats: ['extract'],
  extract: {
    schema: ListingSchema,
    prompt: 'Extract all product listings with their titles, prices, and categories'
  },
  waitFor: 3000,
})

// Clean structured data, regardless of Vue component complexity
result.extract.items.forEach(item => {
  console.log(`${item.title}: $${item.price}`)
})
```

## Handling Vue Transitions and Animations

Vue's `<Transition>` and `<TransitionGroup>` components animate content in and out of the DOM. During animations, elements might be in a partially visible state that gives scrapers inconsistent results.

The fix is to wait longer than the animation duration:

```javascript
// If the Vue app uses entry animations on data load
const result = await app.scrapeUrl('https://vue-portfolio.com/projects', {
  formats: ['markdown'],
  actions: [
    { type: 'wait', milliseconds: 3000 }, // Wait for transitions to complete
  ],
  waitFor: 1000,
})
```

## Monitoring Vue App State Changes

For dashboards that update in real time (Vue's reactive data model makes this common), consider scraping at intervals and diffing results:

```javascript
async function monitorVueDashboard(url, intervalMinutes = 15) {
  let previousData = null

  setInterval(async () => {
    const result = await app.scrapeUrl(url, {
      formats: ['extract'],
      extract: {
        prompt: 'Extract current dashboard metrics and their values'
      },
      waitFor: 3000,
    })

    const currentData = JSON.stringify(result.extract)

    if (currentData !== previousData) {
      console.log('Dashboard updated:', result.extract)
      await saveToDatabase(result.extract)
      previousData = currentData
    }
  }, intervalMinutes * 60 * 1000)
}

monitorVueDashboard('https://analytics-dashboard.com/overview')
```

Vue and Nuxt sites are well-designed applications with predictable rendering patterns. Once you account for the async data-fetching lifecycle, Firecrawl extracts their content as reliably as static HTML.
