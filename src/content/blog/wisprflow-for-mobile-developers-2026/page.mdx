## Where Mobile Devs Lose the Most Documentation Time

Mobile development has more context-switching pressure than almost any other engineering discipline. You're not sitting at one machine. You're moving between:

- Physical test devices (5, 10, sometimes 20+ in a device lab)
- Android Studio or Xcode on your Mac
- Firebase Crashlytics or Sentry dashboards
- TestFlight or Play Console distribution flows
- Simulator/emulator configurations
- Remote test sessions with actual users

Every transition is an opportunity to lose context. And documentation — the thing that preserves context — is always the casualty when context switches are expensive.

Here's where voice capture changes the calculus:

### Device Testing on the Floor

Walking a device lab or testing a retail kiosk implementation, you can't type. Your hands are on hardware. A voice note takes two seconds:

> "Pixel 7a, Android 14, the swipe-to-dismiss gesture on the notification drawer fires the wrong handler when the app is in foreground. Only happens after backgrounding and resuming at least once. Doesn't repro on Pixel 6."

That's a precise bug report, captured immediately, in a format that goes straight into your issue tracker when you're back at your desk.

### API Integration Context

Third-party API integrations generate the kind of knowledge that only exists in your head and vanishes when the sprint ends. The quirks, the undocumented behaviors, the edge cases you discovered the hard way.

> "Firebase Remote Config doesn't respect the minimum fetch interval on first launch if the user installs and immediately opens offline. Workaround: force a local default config with sensible values and only merge remote on successful fetch."

That's tribal knowledge. Voice capture turns it into documentation.

### Cross-Platform Behavioral Differences

iOS and Android handle the same UI patterns differently. Keyboard behavior, safe area insets, back navigation, notification permissions — the list of platform-specific behavior is long and constantly growing as both OS teams ship updates.

Voice notes during cross-platform testing sessions are where this knowledge gets preserved:

> "iOS 17.4 changed the scroll behavior when a keyboard is present. Our login form now pans the wrong direction on iPhone 15 Pro Max. Works fine on the 14 and earlier. Android is unaffected. Check keyboard avoiding view implementation — probably need platform-specific logic."

### Performance Profiling Observations

Profiling sessions are intense focus work. You're watching flame graphs, watching frame rates, correlating allocation spikes with user interactions. Taking detailed typed notes during this breaks the mental model you're building.

Voice capture preserves observations without breaking focus:

> "The image grid causes a 60ms GC pause every time a new row loads. The pause corresponds exactly to loading full-resolution thumbnails. Scaling down to 200px wide on load reduces this to under 5ms. Need to update the thumbnail loading policy across the whole app."
